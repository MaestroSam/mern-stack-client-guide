CONFIGURING OUR PORT


  If you read the docs correctly, you'll have figured out that this happens in angular.json or using  ng serve --port. Basically,
  
  the docs say that any option in ng serve --<option> can be included in the serve block as a property.
  
  So instead of using ng serve --port 8001, I've set "port": 8001 in the serve block in the angular.json file.

  
    "serve": {{'{'}}
    "builder": "@angular-devkit/build-angular:dev-server",
    "options": {{'{'}}
    "browserTarget": "mern-stack-guide-client:build",
    "port": 8001
    {{'}'}},
    ...

WHERE DOES OUR APP START?


  At main.ts
  
  - main.ts imports AppModule.
  - AppModule imports AppComponent.
  - AppComponent selects app-root as it's containing element.
  - AppComponent renders it's template: app.component.html, in <app-root></app-root>.
  - Can you find app-root in index.html?

  
Okay, there's a bit more: css and assets


  In order to access global context css and assets in our index.html or our component templates, we need to tell webpack (read angular) where to find them.

  We do that in the angular.json config file, within the architect block. Take a look:
  
  
      "architect": {{'{'}}
        "build": {{'{'}}
          ...
          "options": {{'{'}}
            ...
            "assets": [
            "src/assets"
            ],
            "styles": [
            "src/styles.css"
            ],
            "scripts": []
      },
  

  Here, we're saying that webpack should include the files src/assets, and src/styles.css in the global context for use in our angular app.
  
  The angluar doc has more on this
